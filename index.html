Subjimandi App Audit

Sidenote: There 2 main navigations: 1. hamburger menu(which wasn't clear visible initially) and a 2.bottom navigation which adds confusion.

The main navigation has 3 sections:

  1. Ordering
  2. Confirmation
  3. Transit

This is the 3 main process every order goes through in our operations. But as an user, they are exploring what's available, then request for quote and accept/decline the order. The order is confirmed and produce is then delivered to them. There needs to be a mapping between these 2 model in our design. The user doesn't know when an order has moved from ordering to confirmation to transit and don't matter for them.
